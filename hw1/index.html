Task 1: Drawing Single-Color Triangles

I implemented my triangle rasterization algorithm in three main steps: checking the winding order of the given points, finding the bounding box of the triangle, and verifying and rasterizing each valid point within the boundaries. The cross product test is used in the helper function is_counter_clockwise to check the triangle’s winding order; a positive output from the function would indicate that the points are given in counter-clockwise order, which suits the algorithm’s logic. If it were to output a negative value signifying a clockwise winding order, the algorithm would swap the latter two points and convert it into counter-clockwise. Before sampling and rasterizing necessary points, the bounding box of the triangle is found by finding the respective min/max of the x and y values of the three given points. They are labeled as the starting and ending points of the for-loop traversal. The algorithm then performs a nested for-loop on each of the points in the bounding box, perturbing the points by 0.5 in the x and y coordinates before passing them into three stacked inside_edge helper function calls that use the line test to determine the location of the point. Should the functions collectively return true, we know that the point must be within the triangle, and we can fill the nearest pixel with color. This algorithm is no worse than sampling every single point within the bounding box because it loops through strict boundaries in the x and y direction, stopping as soon as it reaches the right and bottom edges of the bounding box.

Task 2: Antialiasing by Supersampling

Supersampling enhances image quality by sampling multiple points per pixel and interpolating their colors, essentially taking a high resolution image and shrinking it down to a lower frequency image surface. Here the algorithm store these samples in a sample buffer, which is a 1D array sized at width * height * sample_rate. After sampling, each pixel's color is then determined by averaging across the colors of all its sub-pixel samples. The sample locations within each pixel are evenly distributed in a grid pattern to ensure better coverage. This method reduces aliasing and jagged edges by capturing more color information per pixel, smoothing out high-frequency artifacts and making edges appear more natural. To support supersampling, I modified the pipeline from Task 1 by: 1. Updating sample locations: Each pixel is subdivided into sqrt(sample_rate) x sqrt(sample_rate) grid positions where color samples are taken. 2. Expanding the usage of sample buffer: Instead of storing a single color per pixel, we now store sample_rate colors per pixel. 3. Modifying resolve_to_framebuffer: When transferring colors to the framebuffer, we accumulate all the sub-pixel samples and normalize them to compute the final pixel color. When rasterizing triangles, we test each sub-pixel sample point to determine if it lies inside the triangle. But now with supersampling in effect, even if the point does not fully lie within the triangle, we would still store the color of the portion within the triangle in its corresponding sample. Hence, border pixels that were solely distinguished by solid colors/white can now take on colors with smoother intensities (ex: different shades of pink, as shown in the example below). Later, during the resolve stage, the algorithm averages the stored sample colors for each pixel, reducing sharp edges and improving gradient smoothness.

Task 4: Barycentric coordinates

Barycentric coordinates is a method for linear interpolating values at the three vertices of a triangle, allowing for special effects such as textures, colors, and patterns based on proportions and distances of each point in the interior relative to the vertices. The distance is calculated by comparing the point to the perpendicular line connecting an edge to one vertex. In the example below, we intend to interpolate the three colors (red, blue, green) of the vertices. For any given point (x, y) within the triangle, it will get assigned to a color depending on how close it is proportionally to each of the three vertices; the proportions decide how much each color contributes to the final appearance of the point. The midpoint between the left vertex (red) and the top vertex (blue), for example, appears purple because it is contributed by 0.5x red and 0.5x blue. The overall result is a smoothly applied color gradient that blends as we reach the inner points. In a more general sense, barycentric coordinates can help determine the relative “center of mass” of a 2D triangle and then adjust everything accordingly.

Task 5: "Pixel sampling" for texture mapping

Pixel sampling is a way to interpolate a higher resolution image/texture onto a smaller screen space. In my algorithm I implemented pixel sampling by building off of the barycentric coordinates from Task 4, finding the uv coordinates of each point within the texturized triangles labeled by vertices (u0, v0), (u1, v1), and (u2, v2). With the uv coordinates it can then scale with respect to the dimensions of the Texture mipmap, fetching the corresponding texel color to fill the pixel on screen. Nearest sampling is less computationally costly, as it only samples the nearest texel of each point, but conversely produces a more blocky, pixelated output. Whereas bilinear sampling takes into account the four nearest texels of each point, applying three lerp functions to create a smoother transition between textures. It is clear that the two methods attend to different parts of the picture. Both methods showed improvements with a higher supersampling rate, with bilinear sampling at supersampling rate of 16 per pixel producing the best overall picture that balances both the rough integrity of the curved shape as well as the lines of the texts. From this example we can see that the two methods will produce a large difference when processing areas with high frequency texture details (ex: texts). Nearest sampling would produce blocky artifacts while bilinear sampling could better attend to the slim lines and surfaces. This is because at the border of a letter, nearest sampling may catch a pixel that is outside of the letter and produce an incorrect sample. Bilinear sampling, however, interpolates all four nearest pixels nearby, which is more robust against the changing strokes and ligatures of typeface.

Task 6: "Level Sampling" with mipmaps for texture mapping

Level sampling is a way to approximate the effects of supersampling without the high overhead of parsing through uniformly high quality images. It effectively selects the appropriate mipmap level for a texture based on its screen size. Instead of always using the highest-resolution texture, we downsample (i.e. sample through other levels of the precomputed mipmap) when the texture is “far away” in the image to reduce aliasing and improve performance. This is determined by calculating how much the texture changes per pixel, which is its level of detail; if a pixel’s texture changes a lot (meaning it’s squished and is located further from viewpoint), it means we can safely sample from a higher level mipmap without losing too much quality. For implementation, I computed the pixel’s level of detail using texture coordinate differences and rounded to the nearest mipmap level (power of 2) for nearest level sampling. For linear sampling, instead of restricting to a single level, interpolation is done by calculating a weighted contribution of the samples from two nearby levels. For example, if the calculated level of detail is 1.8, the algorithm will sample from both mipmap level 1 and level 2, with weights 0.2 and 0.8 respectively based on how close the number is to the integer levels. These then get passed in to the sample_nearest or sample_bilinear functions from Task 5 to return the correct Color for rasterizing the triangle. Pixel sampling (two options: nearest, bilinear) is the fastest but can produce aliasing. Level sampling with mipmaps improves performance and smoothness, especially with trilinear filtering, providing a good balance between image quality and computational costs. Supersampling, which takes multiple samples per pixel, offers the highest quality but is computationally expensive because it needs to scale the sample buffer to store all samples and then collectively take averages.
